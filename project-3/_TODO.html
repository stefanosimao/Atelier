Goal

The goal of these exercises is to improve your understanding of JavaScript as an object-oriented programming language for the Web browser.

We assume you know how to program with objects in some other object-oriented programming language.

All the code should be written directly into the template (init.js) provided in the skeleton package. After completing the exercise, make sure all the tests are green and push your changes to your GitHub class repository.

This exercise builds on and extends the code of the previous assignments. You may reuse all functions from the script.js.

Automated tests

Open the test.html page in a Web browser to run the tests automatically. The source of the test cases is found in js/test.js, you are welcome to read them but should not modify them.

Task 1. Basic Functions

In this task you are required to implement some functions. Notice that for each function we give you a JSDoc documentation of its arguments and and return value (these are known as the signature of the function). There are also test cases available to check your code.

While you should be able to program from scratch all of the algorithms required to implement the following functions, it is ok to take advantage of the JavaScript built-in functions to speed up your development efforts. Still, no external modules or libraries are allowed.

  1. Find By

    Implement a find_by function, which given an array of objects, returns an array with the objects which share the same field whose values exactly match the given filter object.

    For example, given the same array of objects as before:

    find_by(a, {"genre": "Blues"}) should return:

    [
  {
    "genre": "Blues",
    "artist": "Marsalis",
    "title": "C"
  }
]

    find_by(a, {"genre": "Jazz", "artist": "Armstrong"}) should return:

    [
  {
    "genre": "Jazz",
    "artist": "Armstrong",
    "title": "A"
  }
]

    find_by(a, {}) should return the original array

    [
  {
    "genre": "Jazz",
    "artist": "Armstrong",
    "title": "A"
  },
  {
    "genre": "Jazz",
    "artist": "Marsalis",
    "title": "B"
  },
  {
    "genre": "Blues",
    "artist": "Marsalis",
    "title": "C"
  }
]

    If no object matches the filter, like with:

    find_by(a, {"album" : "Improv"})

    then an empty array should be returned

    []

    Any undefined or null value of the filter object should be ignored

    find_by(a, {"album" : undefined, "title": "C"})

    results in:

    [
  {
    "genre": "Blues",
    "artist": "Marsalis",
    "title": "C"
  }
]

Task 2. Dynamic song table

Load the js/data.js script which will create a global variable song_data which contains the information to be represented in the songs.html page.

In particular, it should be possible to:

  • Display the entire song table (if no URL parameter are given)
  • Filter the song table by artist (using the ?artist=Name URL query parameter)
  • Filter the song table by album (using the ?album=Name URL query parameter)
  • Filter the song table by genre (using the ?genre=Name URL query parameter)

When displaying the filtered song table, the title of the page should change to reflect the chosen filter. Also indicate how many songs are listed.

When displaying the song table filtered by genre, display the list of artists for the given genre in the sidebar. The sidebar artists should link to their corresponding filtered song table (filtered by artist)

When displaying the song table filtered by artist, display the list of albums for the given artist in the sidebar. The sidebar albums should link to their corresponding filtered song table (filtered by album)

Hint: use the previously implemented find_by function.

Hint: use the standard URL parser and the URLSearchParams.

Task 5. Playlist Object

Rewrite the playlist so that it becomes object-oriented. You may implement it as a class or as a plain object created using the constructor function Playlist.

The Playlist object provides the following methods

* `Playlist(a, repeat)`
* `toJSON()`
* `load(json)`
* `next()`
* `prev()`
* `appendSong(s)`
* `toHTML()`

The Playlist object provides the following public fields: 

* `index`
* `songs`
* `repeat`

  1. Playlist(a, repeat)

    Initialize a playlist from the song URL array a and set it to the initial index position (0).

    The repeat Boolean property determines whether the playlist is supposed to be played multiple times around (true), or only once (false). By default it is false.

  1. toJSON()

    Return the entire state of the playlist object as a JSON string.

  2. load(json)

    Initialize the playlist object from the corresponding JSON string representation.

    If the given json is invalid it should fail silently and preserve the existing state of the object.

  3. next()

    Advance the playlist index position by one song and return the new song URL.

    If the repeat property is set, wrap back to the beginning when running out of songs. Otherwise, simply return undefined and keep the current position so that calling prev() will return the last song again.

  4. prev()

    Return the previous song URL and update the playlist index position accordingly.

    If the repeat property is set, wrap around to the end when running out of songs. Otherwise, simply return undefined and keep the current position so that calling next() will return the first song again.

  5. appendSong(s)

    add the song URL s at the end of the playlist.

  6. toHTML()

    render the content of the playlist as an HTML string

Task 6. Player with Playlist

The Player initialization code should be extended with encapsulate the ability to create the HTML markup necessary to control and display the state of the audio player element and its playlist.

Use the playlist.html page to display the extended Player and its Playlist objects.

If done right, it should be possible to instantiate two or more independent player objects on the same page.

The playlist page should be initialized with this function:

  • init_player_with_playlist(dom, key, a)

More in detail:

  1. __init_player_with_playlist(dom, key, a)__

    Given a placeholder dom element, inject the player HTML markup and the corresponding hidden audio element and setup the necessary event listeners to control the player (as in the previous assignment) from the HTML buttons.

    Create an internal playlist object by loading its state from the given localStorage key. Use the given array of song URL a as the default value, in case the localStorage copy cannot be loaded.

    Visualize the content of the playlist as part of the player markup. Highlight the currently playing song and add a <button> to control the playlist repeat property.

    Connect the next and prev buttons to the corresponding methods of the playlist object.

    Add a new repeat button which can be used to toggle the repeat property of the playlist.

Task 7. Append to Playlist

Add a button next to each song displayed on the songs.html table to append the corresponding song to the playlist stored in localStorage at the same key used for initializing the playlist in the playlist.html page.

If the playlist changes, the player on playlist.html should reload it without interrupting the currently playing song.

Task 8. Remove from Playlist (Optional)

On the playlist.html page, add buttons to remove individual playlist entries and one button to remove all entries with one click.

Task 9. Extract metadata before upload (Advanced, Optional)

Extend the upload.html page so that when the user chooses a .mp3 file:

  • the file size and song duration (in M:SS) are shown on the upload form before the user clicks on the Submit button.

  • the corresponding metadata is automatically entered by parsing the ID3 tag found in the file and used to populate the corresponding input fields.

Even better, instead of using the file chooser input, let the user drag and drop the .mp3 file on the browser window.

Hint: use a library to parse the .mp3 file metadata, such as jsmediatags.
